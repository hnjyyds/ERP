## 五、架构特征

### 5.1 路由

- **无 React Router**，通过 `onNavigate` 回调 + `routes.ts` 路径常量手动路由
- 支持列表路径和详情路径（`/masterdata/products/:id`）

### 5.2 状态管理

- **无 Redux / Zustand / Context**，全部通过 `useState` + props 层层传递
- `App.tsx` 是唯一的全局状态持有者（约 24400 行）

### 5.3 组件复用

- `TradingPartnerPage`：泛型组件，通过 props 注入实现客户/供应商/合作伙伴复用
- `shared/ui/`：Metric、PanelTitle、ColumnViewControl 等轻量复用组件
- 尚未抽取通用 `EntityPageShell` / `FilterToolbar` / `EntityFormModal`

### 5.4 API 层

- 集中在 `api.ts`，基于 `fetch` 封装，支持 JWT Token 认证
- 统一 `ApiResponse<T>` 格式，`ApiError` 类 + 友好错误文案

### 5.5 国际化

- 后端下发 `I18nConfig`，前端 `t()` 函数翻译
- 支持中文 `zh-CN` 和英文 `en-US`

### 5.6 部署架构

- Docker Compose：Nginx（前端静态 + API 代理）→ 双 FastAPI 后端 → SQLite
