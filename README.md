## 建议技术栈

- 前端：React + TypeScript + Vite。当前功能验证页基于 Ant Design，后续 UI 重构统一切换为 Semi Design
- 后端：FastAPI + Python 3.12+，按领域模块组织的模块化单体
- ORM/迁移：SQLAlchemy 2.0；Alembic 后续接入
- 数据校验：Pydantic v2
- 数据库：当前开发环境使用 SQLite；生产建议 PostgreSQL
- 缓存/任务：Redis + Celery/RQ/APScheduler 后续接入
- 文件存储：MinIO 或兼容 S3 的对象存储
- 部署：Docker Compose 起步，后续可演进到 Kubernetes

## 首期目标

首期目标是建设一套面向外贸企业的业务管理系统，覆盖基础资料、样品、销售、采购、采购跟单、QC、单证、财务、仓库、审批、提醒、报表分析等核心流程。

优先采用模块化单体架构，先保证业务闭环、数据一致性和交付效率，再根据实际并发、组织规模和集成需求拆分服务。
