# 开发步骤

## 阶段 0：需求确认与项目基线

目标：把 PDF 需求转化为可执行范围，确认首期交付边界。

- 梳理角色：业务员、业务主管、采购、单证、仓库、QC、财务、经理、系统管理员。
- 确认业务编号规则：报价单、出口合同、采购合同、出货单、入库单、出库单、付款单等。
- 确认审批流程：哪些单据需要审批、审批层级、驳回规则、撤回规则。
- 确认报表口径：金额、数量、已出/未出、已采/未采、应收/应付、毛利。
- 输出首版 PRD、字段清单、流程图和验收标准。

## 阶段 1：技术底座

目标：先把可持续开发的工程基础搭起来。

- 初始化前端、后端、数据库、部署脚本。
- 前端统一采用 React + TypeScript + Vite。当前功能验证页面基于 Ant Design，后续 UI 重构统一切换为 Semi Design；既有 Vue 原型仅作为迁移参考。
- 建立 FastAPI 分层骨架：APIRouter、Pydantic schema、Service、Repository/DAL、SQLAlchemy model、Unit of Work。
- 建立 Contract-first TDD 流程：先定义 API/Schema/Service/DAL/架构契约，再写测试和实现。
- 建立全异步数据库会话：FastAPI `Depends` 注入 async session，Repository/DAL 禁止自行提交事务。
- 建立登录、用户、组织、角色、权限、菜单。
- 建立通用单据状态机：草稿、审批中、已审批、驳回、作废、关闭。
- 建立审批流、待办、消息提醒、日程、操作日志。
- 建立附件、图片、导入导出、模板打印、枚举字典。
- 建立 CI、代码规范、单元测试、集成测试、架构约束测试和质量门禁。

## 阶段 2：基础资料

目标：完成所有业务单据依赖的主数据。

- 商品资料、图片、海关编码、税率、包装。
- 商品配件明细和单位耗料。
- 客户、客户联系人、信用等级、授信额度。
- 供应商、供应商联系人、信用等级、授信额度。
- 合作伙伴、单证资料、常用地址、币种、贸易条款。
- 主数据导入导出和模糊查询。

## 阶段 3：样品管理

目标：让确认样、大货样能够支撑采购跟单节点。

- 打样管理：打样要求、费用、进度。
- 样品登记：样品分类、来样日期、确认样/大货样提交日期。
- 寄样管理：寄样记录、费用、审核、统计。
- 样品记录与客户、供应商、商品、采购合同关联。

## 阶段 4：销售业务

目标：建立业务源头，从报价到出口合同再到出货计划。

- 出口报价、审批、报价模板、历史报价。
- 报价确认后生成出口合同。
- 出口合同审批、回签、预收款关联。
- 出货明细，从已审批出口合同生成待出运计划。
- 支持多个出口订单一次出运。

## 阶段 5：采购业务与采购跟单

目标：完成从销售合同到采购执行的闭环。

- 采购询价和供应商报价。
- 从出口合同和商品配件明细生成采购计划/采购合同。
- 支持库存采购、多个出口合同合并采购合同。
- 采购合同审批、供应商交货提醒、付款提醒。
- 跟单流程模板：节点、标准天数、提前提醒天数。
- 采购合同确立后生成节点计划。
- 采购合同进度查询表。

## 阶段 6：QC 与仓库

目标：把 QC、入库、库存、出库接入采购履约进度。

- QC 查验：查验日期、结果、异常、附件。
- 入库计划：根据采购合同交货期生成。
- 货物入库：待检入库、正式入库、审批、入库台账。
- 出库计划：根据领料、发料、发货计划生成。
- 货物出库：正式出库、异常出库、出库台账。
- 库存调拨、库位管理、库存查询、负库存控制。
- QC、入库、出库实际日期回写采购跟单。

## 阶段 7：单证管理

目标：减少重复录入，支持报关、结汇和进仓通知。

- 信用证登记、交单提醒、有效期提醒。
- 报关单证，从出货明细复制数据生成。
- 结汇单证，从出货明细或报关单证生成。
- 进仓通知，按供应商生成进仓要求。
- 客户索赔和处理记录。
- 单证模板导出 PDF/Excel。

## 阶段 8：财务与利润核算

目标：形成应收、应付、费用、退税、单票毛利闭环。

- 收款管理：银行水单、认领、分摊。
- 付款管理：供应商发票、付款申请、审批。
- 付费管理：合作伙伴费用发票、付款申请。
- 核销退税：核销单、回单、核销、退税状态。
- 杂费分摊和财务结算锁定。
- 单票利润核算和手工关联其他成本。

## 阶段 9：报表分析

目标：让经理能直接看到审批、合同、出运、应收应付和进度风险。

- 审批查询。
- 出口合同统计、采购合同统计、报关出运统计。
- 应收款、应付款、核销单使用情况。
- 业务员出货同期对比、月度出货情况。
- 客户出货同期对比。
- 采购合同进度查询和逾期预警。

## 阶段 10：测试、迁移和上线

目标：保障真实业务可用。

- 单元测试、集成测试、端到端测试。
- 权限和审批流回归测试。
- 报表口径核对。
- 历史数据导入模板和试导入。
- UAT 验收、性能压测、备份恢复演练。
- 生产部署、监控、日志、告警。

## 建议里程碑

| 里程碑 | 范围 | 交付物 |
| --- | --- | --- |
| M1 | 技术底座 + 基础资料 | 可登录、可配置权限、可维护主数据 |
| M2 | 样品 + 销售 + 采购 | 出口合同到采购合同闭环 |
| M3 | 跟单 + QC + 仓库 | 采购进度、入库、出库、库存闭环 |
| M4 | 单证 + 财务 | 单证、收付款、费用、退税、利润闭环 |
| M5 | 报表 + 上线 | 统计分析、数据迁移、UAT、生产部署 |
