# 实现步骤（PDF 详细拆分）

本文档根据 `docs/reference/远景外贸业务管理系统产品介绍-含仓库管理.pdf` 拆分具体实现步骤。实现时默认遵循 [架构设计](03-架构设计.md) 中的 FastAPI 分层、Contract-first TDD、全异步边界、Repository/DAL 稳定契约和质量门禁。

## 使用方式

- `PDF 对照`：对应原 PDF 章节或第 49 页定制开发说明。
- `页面`：前端需要实现的主要页面或弹窗。
- `后端`：FastAPI 模块、核心模型、服务和后台任务。
- `规则`：必须落地的业务规则。
- `测试`：优先补充的自动化测试。
- `验收`：业务侧可以用来确认是否完成的口径。

## 实施总顺序

| 阶段 | 名称 | 目标 |
| --- | --- | --- |
| 0 | 需求和口径确认 | 将 PDF 内容转成字段、流程、权限、报表口径 |
| 1 | 工程和通用底座 | 建立 FastAPI、React、数据库、权限、审批、提醒、附件、模板 |
| 2 | 业务资料 | 商品、客户、供应商、合作伙伴、单证资料、配件明细 |
| 3 | 样品管理 | 打样、样品登记、寄样、样品报表 |
| 4 | 出口业务 | 出口报价、出口合同、出货明细、销售报表 |
| 5 | 采购业务和跟单 | 采购询价、采购合同、开票通知、跟单流程、进度查询 |
| 6 | QC 和仓库 | QC 查验、入库计划、入库、出库计划、出库、调拨、库存 |
| 7 | 单证管理 | 信用证、报关、结汇、进仓通知、客户索赔 |
| 8 | 财务和利润 | 收款、付款、付费、核销退税、杂费、结算、利润核算 |

| 10 | 联调、迁移、上线 | 端到端验收、数据迁移、性能、安全、备份、发布 |

## 阶段 1：工程和通用底座

### 1.1 FastAPI 工程骨架

PDF 对照：系统整体需要网络版、权限、审批、提醒、报表、导入导出等通用能力。

页面：

- 健康检查页或开发调试页。
- 登录页。
- 后台主布局：左侧菜单、顶部用户区、多标签工作区。

后端：

- 初始化 `backend/app`。
- 建立 `app/main.py`。
- 建立 `app/api/v1/router.py`。
- 建立 `app/api/deps.py`。
- 建立 `app/core/config.py`、`app/core/security.py`、`app/core/logging.py`。
- 建立 `app/db/session.py`、`app/db/base.py`。
- 建立统一响应 `ApiResponse[T]`。
- 建立分页 `PageResult[T]`。
- 建立业务异常和错误码。

规则：

- 所有新增路由显式声明 `response_model=ApiResponse[...]`。
- APIRouter 不直接导入 Repository/DAL 或 ORM model。
- Service 负责业务编排、事务、权限、后台任务。
- Repository/DAL 只负责数据库访问。
- 新增路由、Service、DAL 默认 async。

测试：

- `GET /api/v1/health` API 测试。
- 路由注册测试。
- 跨层导入架构测试。
- `response_model` 架构测试。

验收：

- 后端可启动。
- OpenAPI 文档可打开。
- 健康检查返回成功。
- 质量门禁命令可运行：pytest、ruff、类型检查。

### 1.2 前端工程骨架

PDF 对照：系统支持多个页面并列、快捷方式到桌面、工作界面快速切换。

技术路线：前端统一改为 `React + TypeScript + Vite`。当前功能验证页面已基于 Ant Design 跑通核心模块，后续 UI 重构统一切换为 Semi Design，具体页面结构见 `docs/07-SemiDesign前端UI重构设计.md`。既有 Vue 页面作为交互和字段参考，后续新功能和迁移功能均落到 React 目录结构、React 组件和 React E2E。

页面：

- 登录页。
- 工作桌面。
- 主布局。
- 菜单页。
- 多标签页工作区。
- 模块占位页。

后端：

- 提供当前用户、菜单、权限接口。

规则：

- 菜单由后端权限控制。
- 页面按钮基于权限控制显示。
- 列表页统一分页、筛选、排序、列配置。
- 单据页统一包含基本信息、明细、附件、审批记录、操作日志。

测试：

- 前端路由守卫测试。
- API client 错误处理测试。
- React 登录、权限菜单和工作桌面 E2E。

验收：

- 登录后进入工作桌面。
- 不同角色看到不同菜单。
- 页面切换不丢失工作区状态。

### 1.3 数据库、迁移和通用字段

PDF 对照：所有业务模块共享查询、统计、审批、附件、导入导出能力。

页面：

- 无独立业务页面。

后端：

- PostgreSQL。
- SQLAlchemy async engine。
- Alembic 迁移。
- 通用字段 mixin：`id`、`tenant_id`、`created_at`、`created_by`、`updated_at`、`updated_by`、`deleted_at`、`version`。
- 编号规则表。
- 附件表。
- 操作日志表。
- 字典表。

规则：

- 所有核心业务表带通用字段。
- 金额和数量使用 Decimal。
- 软删除不影响唯一约束时，需要设计局部唯一索引。
- 所有表结构变更必须有 Alembic 迁移。

测试：

- 迁移升级测试。
- 通用字段自动填充测试。
- 编号生成并发测试。

验收：

- 数据库可从空库迁移到最新版本。
- 单据编号可按规则生成且不重复。

## 阶段 10：端到端联调、迁移和上线

### 10.1 端到端联调

端到端主流程：

1. 维护商品、商品配件、客户、供应商、合作伙伴、单证资料。
2. 登记样品或打样记录。
3. 创建出口报价。
4. 报价审批并确认生成出口合同。
5. 出口合同审批通过。
6. 根据出口合同和商品配件生成采购合同。
7. 采购合同审批并确立。
8. 自动生成采购跟单节点。
9. 登记确认样、大货样。
10. 登记 QC 查验。
11. 根据采购合同交货期生成入库计划。
12. 货物待检入库和正式入库。
13. 根据发货计划生成出库计划。
14. 正式出库。
15. 从出货明细生成报关单证。
16. 生成结汇单证和进仓通知。
17. 登记收款、付款、费用。
18. 核销退税。
19. 财务结算并查看利润核算。
20. 经理查看进度查询和统计报表。

测试：

- Playwright 或 API E2E 覆盖主流程。
- 每个关键单据状态都断言。
- 每个跨模块回写都断言。

验收：

- 主流程完整跑通。
- 每一步有单据、状态、日志、审批记录和数据权限。
