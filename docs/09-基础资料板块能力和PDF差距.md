# 基础资料板块能力和 PDF 差距

> 盘点日期：2026-06-17  
> 对照来源：`docs/reference/远景外贸业务管理系统产品介绍-含仓库管理.pdf` 中 `1.5.2 业务资料`，以及当前代码、测试和功能检查记录。

## 结论

基础资料板块已经达到“可维护、可查询、可被后续业务引用”的主干状态，但还不能判定为 PDF 意义上的完全完成。

已完成的核心范围包括：商品资料、客户资料、供应商资料、合作伙伴、单证资料的列表查询、新增、编辑、停用或状态维护、联系人或配件明细维护、权限控制、数据权限过滤、前端页面和自动化测试。当前补齐后，客户、供应商、合作伙伴的联系人已经支持完整新增、编辑、删除；合作伙伴也支持停用。

仍未完全闭环的部分集中在：所有下游业务单据的自动带出与快速引用、完整交易/费用记录来源、可配置字典、单证资料在真实报关/结汇/信用证页面中的嵌入式选择，以及部分高级导入导出和视图管理体验。

## 当前能做什么

### 商品资料

- 维护商品基础信息：编号、中英文名称、规格、型号、单位、海关编码、退税率、包装信息、图片地址、状态。
- 查询商品列表，支持关键字检索和列表列显示配置。
- 新增、编辑、停用商品。
- 维护商品配件明细：新增、编辑、删除配件；记录单位耗料、计量单位、默认供应商、采购拆分规则。
- 上传商品图片并回填图片地址。
- 商品导入：支持 CSV/XLSX 文件解析，按结果写入或更新商品。
- 商品导出：支持导出商品 CSV 内容。
- 为后续采购合同从出口合同商品拆分配件采购明细提供数据基础。

### 客户资料

- 维护客户基础信息：编号、中英文名称、国家地区、地址、网站、状态。
- 查询客户列表，支持编号、名称、国家、联系人等模糊查询，支持国家和信用等级过滤。
- 新增、编辑、停用客户。
- 维护联系人：新增、编辑、删除联系人，并保证主要联系人唯一。
- 维护信用资料：信用等级、授信额度、币种、账期、风险备注。
- 按权限返回授信额度等敏感字段；没有信用查看权限时额度为空。
- 查看交易记录入口；当前聚合出口合同和出货计划来源。
- 前端支持列表、筛选、详情、弹窗新增/编辑、联系人编辑/删除和列显示配置。

### 供应商资料

- 维护供应商基础信息：编号、中英文名称、国家地区、地址、网站、状态。
- 查询供应商列表，支持编号、名称、国家、联系人等模糊查询，支持国家和信用等级过滤。
- 新增、编辑、停用供应商。
- 维护联系人：新增、编辑、删除联系人，并保证主要联系人唯一。
- 维护信用资料：信用等级、授信额度、币种、账期、风险备注。
- 按权限返回授信额度等敏感字段。
- 查看交易记录入口；当前聚合采购合同和供应商发票来源。
- 前端支持列表、筛选、详情、弹窗新增/编辑、联系人编辑/删除和列显示配置。

### 合作伙伴

- 维护合作伙伴基础信息：编号、中英文名称、类型、国家地区、地址、网站、状态。
- 合作伙伴类型覆盖：快件公司、货代公司、保险公司、运输公司。
- 查询合作伙伴列表，支持关键字和类型过滤。
- 新增、编辑、停用合作伙伴。
- 维护联系人：新增、编辑、删除联系人，并保证主要联系人唯一。
- 查看费用记录入口；当前聚合合作伙伴费用发票和付费申请来源。
- 前端支持列表、筛选、详情、弹窗新增/编辑、联系人编辑/删除、停用和列显示配置。

### 单证资料

- 维护常用单证抬头资料：收货人、通知人、开证行、提单通知人。
- 维护资料名称、客户标识、国家、地址、联系人、邮箱、电话、银行名称、SWIFT、账号、备注、默认项和状态。
- 查询单证资料列表，支持类型、客户标识和关键字过滤。
- 新增、编辑、停用单证资料。
- 支持同一用户、客户、资料类型下默认项唯一替换。
- 提供 `lookup` 快速引用接口，供报关、结汇、信用证等后续模块按客户和类型取有效资料。
- 前端支持列表、筛选、详情、弹窗新增/编辑、默认项维护和快速引用预览。

### 通用能力

- 基础资料菜单已接入权限体系，不同角色只能看到授权菜单。
- 后端保持 router/service/repository/schema 分层，业务规则在 service 层，数据库访问在 repository 层。
- 基础资料按当前用户或 `view_all` 权限做数据范围控制。
- 停用采用状态变更，不硬删除主数据，便于保留历史业务引用。
- 主要接口都有 API 和 service 测试覆盖，前端有构建检查和 E2E 脚本。

| 合作伙伴费用记录 | 部分完成 | 当前聚合费用发票、付费申请；还需接入单证、物流、保险、运输等来源的全量费用轨迹 |
| 单证资料快速调用 | 部分完成 | 已有 lookup 接口和页面预览；还需在信用证、报关、结汇、提单等真实单证页面嵌入选择弹窗和自动带出 |
| 下游单据默认带出联系人 | 部分完成 | 主联系人数据已具备；仍需在报价、合同、采购、单证、财务等页面逐一接入默认带出和可覆盖逻辑 |
| 类型和状态字典维护 | 部分完成 | 合作伙伴类型、单证资料类型目前是固定枚举；还需后台字典管理页面和接口 |
| 主数据被引用后的限制规则 | 部分完成 | 已采用停用保留历史；还需在被引用后限制关键字段修改、展示引用来源、提供变更日志 |
| PDF 中“业务资料”与后续模块联动 | 部分完成 | 基础资料自身可用，但与销售、采购、单证、仓库、财务的所有自动引用还没有全部走通 |
