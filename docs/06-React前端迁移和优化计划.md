# React 前端迁移和优化计划

## 目标

将既有 Vue 原型迁移为 `React + TypeScript + Vite` 前端。当前功能验证阶段已用 Ant Design 跑通核心页面和 E2E；后续 UI 重构阶段统一切换为 `Semi Design`，实现一套适合外贸 ERP 高频操作的工作台：轻量、密集、可扫描、权限清晰，并按 PDF 功能点逐步补齐页面、交互和端到端测试。

## 迁移原则

- 后端 API 契约不随前端迁移破坏，继续使用 `ApiResponse[T]`。
- 旧 Vue 原型保留在 `frontend/src/legacy/App.vue` 作为字段和交互参考，不参与当前构建。
- 每次只迁移一个可验收的功能面，完成后补充或更新 E2E。
- 迁移期间保留业务字段和状态口径，UI 结构可优化但不能删减业务动作。
- React 页面优先复用通用布局、筛选条、数据表、详情区和表单动作区。
- 列表、单据、审批、库存和报表页面采用紧凑 B 端设计，避免营销化页面。

## UI 组件库路线

- 功能验证期已经用 `Ant Design` 完成大部分页面、接口联调和 E2E，历史检查记录继续保留该实现证据。
- UI 重构期切换到 `Semi Design`，目标不是简单替换组件，而是按页面类型重做信息结构。
- 详细页面结构、每个页面包含内容和实现方式见 [Semi Design 前端 UI 重构设计](07-SemiDesign前端UI重构设计.md)。
- 页面风格按 `.impeccable.md` 固定为外贸 ERP 工作台：浅色、密集、安静、可扫描，状态色只服务于风险、逾期、审批和完成状态。
- 页面类型必须区分：主数据、单据、流程、仓库、财务、报表不能继续复制同一个“列表 + 右侧表单”模板。
- 当前 API client 和后端 `ApiResponse[T]` 契约保持不变，UI 重构只改 React 组件、布局和交互组织。

## 第一阶段

- 建立 React/Vite 工程骨架。
- 实现登录页、Token 保存、当前用户和权限菜单加载。
- 实现主布局、左侧菜单、顶部用户区和工作桌面。
- 保持财务角色只看到工作桌面和财务管理。
- `/finance` 已接入财务管理和利润概览、银行水单、认领分摊、应收账款查询、供应商发票、付款申请、付款审批、应付账款查询、合作伙伴费用发票、付费申请、付费审批、应付费用查询、核销单、报关回单、核销登记、退税登记、核销单使用情况查询、杂费项目配置、单票杂费分摊、杂费分摊查询、财务结算锁定、手工成本关联和利润核算表。
- `/reporting` 已接入经理查询和审批聚合，支持业务主管按单据类型、审批状态、申请人和时间查询待审批/已审批单据；同时接入统计分析报表，支持按日期、状态、客户、供应商、业务员查看出口合同、采购合同、客户出货和业务员月度出货。
- 运行 `npm run build` 和 `node scripts/e2e-auth.mjs`。

## 第二阶段

- 迁移基础资料：商品、客户、供应商、合作伙伴、单证资料。
- 每个页面保留列表、搜索、新增、编辑、明细和联系人/配件等核心能力。
- 每迁移一个模块更新对应 `docs/checks/*` 检查记录。

## 第三阶段

- 迁移样品、销售、采购、采购跟单、QC、仓库模块。
- 优先迁移已经完成后端契约和 E2E 的模块。
- 仓库模块继续覆盖入库计划、入库、出库计划、出库、调拨和库存报表。

## 验收口径

- React 前端可以独立构建。
- 登录、权限菜单和至少一个业务模块有稳定 E2E。
- 每个迁移模块都有功能检查记录，说明页面、API、规则、自动化证据和剩余风险。
- 浏览器实际页面无阻塞性前端错误，关键文字和控件在桌面宽度下不重叠。
