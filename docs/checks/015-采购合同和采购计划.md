## 本次实现范围

后端：

- `GET /api/v1/purchase/contracts`：采购合同列表，支持文本、审批状态、供应商、来源类型过滤。
- `POST /api/v1/purchase/contracts`：新增采购合同，支持库存采购、手工采购和带来源的采购合同。
- `PUT /api/v1/purchase/contracts/{contract_id}`：编辑草稿采购合同，替换合同抬头和商品明细。
- `GET /api/v1/purchase/contracts/{contract_id}`：采购合同明细。
- `POST /api/v1/purchase/contracts/generate-from-export-contracts`：从一份或多份已审批出口合同生成采购合同。
- `POST /api/v1/purchase/contracts/{contract_id}/submit`：提交采购合同审批。
- `POST /api/v1/purchase/contracts/{contract_id}/approve`：审批通过并回写出口合同已采购数量。
- `GET /api/v1/purchase/contracts/reminders`：采购合同付款/交货提醒列表。
- SQLAlchemy async model：`PurchaseContract`、`PurchaseContractLine`、`PurchaseContractSourceLink`、`PurchaseContractReminder`。
- Repository 返回 typed row，封装合同、明细、来源、提醒和统计刷新。
- Service 负责权限校验、已审批出口合同校验、商品配件读取、多合同来源聚合、库存采购、提醒生成、审批状态流转和出口合同采购进度回写。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 采购合同菜单权限：`purchase:contract:view`、`edit`、`view_all`、`approve`。

前端：

- 左侧菜单增加“采购合同”。
- 采购合同页支持列表、搜索、审批状态筛选、来源类型筛选、供应商筛选。
- 支持库存/手工采购合同新增、草稿载入编辑、保存草稿编辑。
- 支持从一份或两份已审批出口合同生成采购合同，按商品配件单位耗料汇总采购数量。
- 明细区展示合同抬头、付款条款、备注、合同商品明细、来源出口合同、付款/交货提醒、进度概览和全局采购提醒。
- 支持提交审批和审批通过；审批通过后回写出口合同采购进度。
- 业务主管可访问采购合同；财务角色无采购合同菜单且接口返回 `403`。
- React 回归修复：生成、保存、提交、审批后使用接口响应对象稳定刷新当前详情；当长期 E2E 数据超过列表默认 `50` 条时，新建/编辑合同不会被分页列表刷新挤出详情区。

## 可用性检查

- 采购合同列表、筛选、库存采购表单、出口合同生成表单和明细审批在同一工作区。
- 来源生成表单明确支持两份出口合同，适合常见多合同合并采购。
- 合同明细展示付款条款、备注、来源合同、商品明细、提醒和进度，主管审批时不用跨页核对。
- 库存采购与出口合同采购共用合同结构，但来源类型清晰可见，避免混淆。
- 草稿编辑、提交审批、审批通过按钮按状态启用，减少误操作。
- 财务角色不会看到采购合同菜单，避免越权入口。
