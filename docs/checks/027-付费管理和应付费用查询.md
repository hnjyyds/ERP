## 本次实现范围

后端：

- `POST /api/v1/finance/partner-fee-invoices`：财务登记合作伙伴费用发票，可关联合作伙伴、出运单和业务员。
- `GET /api/v1/finance/partner-fee-invoices`：财务按发票号、合作伙伴、出运单、业务员、费用类型和付款状态查询费用发票。
- `POST /api/v1/finance/fee-payment-requests`：财务基于合作伙伴费用发票创建付费申请。
- `GET /api/v1/finance/fee-payment-requests`：财务查询付费申请列表，支持状态、费用类型、合作伙伴、业务员和出运单过滤。
- `POST /api/v1/finance/fee-payment-requests/{fee_payment_request_id}/approve`：财务审批并登记付费金额，生成付费分摊。
- `GET /api/v1/finance/fee-payables`：财务查询合作伙伴应付费用、已付费和应付余额。
- 新增 `modules/finance/fee_payments` 分层模块：model、schema、repository、service、provider。
- 启动时注册付费 ORM 模型，保持 FastAPI controller 只做 HTTP 协调和异常转换。

前端：

- `/finance` 在付款工作区下新增付费工作区。
- 合作伙伴费用发票列表支持关键字、状态、费用类型、合作伙伴和出运单筛选，支持行选择和状态标签。
- 费用发票登记表单支持费用发票号、发票日期、合作伙伴、合作伙伴类型、出运单、业务员、费用类型、金额、币种、到期日和备注。
- 付费申请和审批区展示当前费用发票已付/未付金额，支持新增付费申请和审批付费。
- 付费申请列表展示申请号、状态、费用类型、合作伙伴、出运单、申请金额和已付金额。
- 应付费用查询支持按关键字、状态、费用类型、合作伙伴和出运单号筛选，展示发票金额、已付费、应付余额和付款状态。
- 移动端付费区跟随收款/付款区单列布局，避免页面级横向溢出。
