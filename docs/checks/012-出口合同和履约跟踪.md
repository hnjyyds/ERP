## 本次实现范围

后端：

- `GET /api/v1/sales/contracts`：出口合同列表，支持文本、审批状态、客户过滤。
- `POST /api/v1/sales/contracts`：新增出口合同，包含合同抬头和商品明细。
- `PUT /api/v1/sales/contracts/{contract_id}`：草稿合同编辑，替换抬头和商品明细。
- `GET /api/v1/sales/contracts/{contract_id}`：合同明细。
- `POST /api/v1/sales/contracts/{contract_id}/submit`：提交审批。
- `POST /api/v1/sales/contracts/{contract_id}/approve`：审批通过并发布 `ExportContractApproved` 事件。
- `POST /api/v1/sales/contracts/{contract_id}/signature`：登记客户回签信息。
- `POST /api/v1/sales/contracts/{contract_id}/advance-payments`：关联预收款。
- `GET /api/v1/sales/contracts/{contract_id}/export?format=pdf|excel`：合同 PDF/Excel 导出预览。
- `POST /api/v1/sales/quotations/{quotation_id}/confirm-contract`：报价确认后落入正式 `ExportContract` 草稿。
- SQLAlchemy async model：`ExportContract`、`ExportContractLine`、`ExportContractSignature`、`ExportContractAdvancePayment`、`ExportContractEvent`。
- Repository 返回 typed row。
- Service 负责权限校验、草稿编辑规则、审批状态流转、回签、预收款、采购/出货统计、导出和事件发布。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 出口合同菜单权限：`sales:contract:view`、`edit`、`view_all`、`approve`、`export`。

前端：

- 左侧菜单增加“出口合同”。
- React 出口合同页支持列表、搜索、审批状态筛选、客户筛选。
- 支持新增合同、载入草稿、保存草稿编辑、提交审批、审批通过、登记回签、关联预收款、导出 PDF/Excel。
- 明细区展示合同抬头、合同商品明细、商品图片、历史询报价参考、合同采购情况、合同出货情况、预收款记录、回签记录和导出预览。
- 业务主管可访问出口合同；财务角色无出口合同菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，适合业务员从合同录入推进到回签、收款、采购和出货跟踪。

## 可用性检查

- 合同列表、筛选、新增表单和明细在同一工作区，业务员可以边查边录。
- 合同抬头、付款条款、备注、客户回签和预收款集中展示，减少跨页核对。
- 合同商品明细展示图片、规格型号、数量、单价和金额，方便核对订单内容。
- 历史询报价参考紧跟商品明细，便于比较同客户同商品的历史价格。
- 合同采购情况和出货情况直接显示已完成/未完成数量和状态，为后续采购、出货模块提供入口。
- 草稿编辑、提交审批、审批通过按钮按状态启用，减少误操作。
- 财务角色不会看到出口合同菜单，避免越权入口。
