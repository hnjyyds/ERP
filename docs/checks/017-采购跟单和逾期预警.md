## 本次实现范围

后端：

- `GET /api/v1/followup/templates`：跟单模板列表，首次访问自动生成标准采购跟单流程。
- `POST /api/v1/followup/templates`：新增跟单模板和节点。
- `PUT /api/v1/followup/templates/{template_id}`：编辑模板节点标准天数和提前提醒天数。
- `GET /api/v1/followup/plans`：采购合同跟单计划列表，支持文本、状态、供应商、采购合同过滤。
- `POST /api/v1/followup/plans/from-purchase-contract`：从已审批采购合同生成跟单计划。
- `GET /api/v1/followup/{plan_id}`：跟单计划明细。
- `POST /api/v1/followup/sample-events`：同步样品登记产生的确认样/大货样跟单事件。
- `POST /api/v1/followup/source-events`：消费 QC、入库、出库等来源业务事件，回写实际日期。
- `GET /api/v1/followup/overdue-nodes`：按扫描日期计算逾期节点。
- 采购合同审批通过后自动生成跟单计划。
- SQLAlchemy async model：`FollowProcessTemplate`、`FollowProcessNode`、`PurchaseFollowPlan`、`PurchaseFollowNode`。
- Repository 返回 typed row，封装模板、计划、节点、逾期扫描。
- Service 负责权限校验、默认模板、计划生成、来源事件回写、逾期扫描和数据权限。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 权限：`followup:template:view/edit`、`followup:plan:view/edit/view_all`。

前端：

- “采购跟单”菜单接入正式页面。
- 支持跟单计划列表、状态筛选、供应商筛选、采购合同筛选。
- 支持维护默认跟单模板的 6 个节点标准天数和提前提醒天数。
- 支持按采购合同 ID 生成跟单计划。
- 明细区展示采购合同节点预计日期、提醒日期、实际日期、状态和来源。
- 支持同步样品事件，回写确认样/大货样节点。
- 支持通过来源事件回写 QC、入库、出库节点。
- 支持按扫描日期查看逾期节点预警。

  - 通过 API 创建采购合同，提交并审批。

  - 通过 API 创建确认样记录，样品登记产生跟单事件。

  - 等待 `/api/v1/followup/plans` 返回。
  - 按采购合同号筛选计划。
  - 校验 6 个默认节点可见。
  - 点击“同步样品事件”，确认样节点显示实际日期。
  - 在页面回写 QC 查验来源事件，QC 节点显示实际日期。
  - 按 `2026-09-05` 扫描逾期，校验入库等逾期节点显示。

## 可用性检查

- 计划列表、模板配置、来源回写、节点进度和逾期预警集中在同一工作区。
- 节点表按预计日期、提醒日期、实际日期、状态和来源排列，适合采购人员逐票核查。
- 状态文案使用“待完成、已完成、已逾期”，减少内部状态码暴露。
- 来源列显示样品事件、QC 查验、入库单、出库单，明确实际日期不是重复手填。
- 逾期扫描日期可调，便于主管做日常预警和历史复盘。
