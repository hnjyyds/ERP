## 本次实现范围

后端：

- `GET /api/v1/warehouse/inbound-plans`：入库计划列表，支持文本、入库类型、状态、供应商、采购合同过滤。
- `POST /api/v1/warehouse/inbound-plans/from-purchase-contract`：从已审批采购合同生成入库计划。
- `GET /api/v1/warehouse/inbound-plans/{plan_id}`：查看入库计划明细。
- `POST /api/v1/warehouse/inbound-plans/{plan_id}/schedule`：保存仓库和库位预安排，状态变为 `scheduled`。
- 采购合同审批通过后自动生成采购跟单计划和入库计划。
- 入库类型包括：配料、采购、加工、生产、包装、灭菌。
- SQLAlchemy async model：`InboundPlan`、`InboundPlanLine`。
- Repository 返回 typed row，封装列表筛选、合同去重、明细和排库位。
- Service 负责权限、采购合同审批校验、数据范围、状态流和响应组装。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 权限：`warehouse:inbound_plan:view/edit/view_all`。

前端：

- 左侧菜单增加“入库计划”。
- 页面支持入库计划列表、状态筛选、类型筛选、供应商筛选、采购合同筛选。
- 支持从采购合同 ID 手动生成入库计划。
- 支持为选中计划保存仓库、库位、经办人和计划入库日。
- 明细区展示采购合同、供应商、类型、状态、计划入库日、库位和待入库商品。
- 采购合同审批成功提示已生成跟单和入库计划。

React 迁移补充：

- `frontend/src/App.tsx` 新增 `/warehouse/inbound-plans` React 路由和 `InboundPlansPage`，替代占位模块。
- 复用 Ant Design 表格、表单、按钮、提示和现有 ERP 工作台布局。
- 新增入库计划状态、入库类型、明细状态中文标签映射。
- 新增从采购合同生成计划和保存库位预安排的 payload 转换。
- 页面保留 E2E 依赖的字段 ID：`inbound-plan-contract-id`、`inbound-plan-type`、`inbound-plan-date`、`inbound-schedule-date`、`inbound-warehouse-id`、`inbound-location-name` 等。
- `frontend/src/styles.css` 新增入库计划概览条、双栏工作区、表格横向保护和移动端适配。

## 可用性检查

- 列表、生成、排库位和明细在一个工作区内完成。
- 状态和类型均以中文展示，保留后端稳定状态码。
- 计划日期默认来自采购合同交货日期，仓库可以按实际情况提前或调整。
- 选中计划后排库位表单自动带出当前计划日期和已有库位。
- 页面在 in-app 浏览器打开 `http://127.0.0.1:5173/warehouse/inbound-plans` 可见菜单、标题、列表、生成表单、排库位表单和详情，控制台无前端错误。
- React 浏览器验收：桌面 `1280px` 宽和移动 `390px` 宽均可见标题、列表、生成/排库位表单和详情，无横向溢出，控制台无 error。
