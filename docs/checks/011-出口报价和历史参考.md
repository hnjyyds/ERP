## 本次实现范围

后端：

- `GET /api/v1/sales/quotations`：出口报价列表，支持文本、审批状态、客户过滤。
- `POST /api/v1/sales/quotations`：新增出口报价，包含商品明细、规格型号、销售价、货运方式、运费和采购参考价。
- `PUT /api/v1/sales/quotations/{quotation_id}`：草稿报价编辑，替换抬头和商品明细。
- `GET /api/v1/sales/quotations/{quotation_id}`：报价明细。
- `POST /api/v1/sales/quotations/{quotation_id}/submit`：提交审批。
- `POST /api/v1/sales/quotations/{quotation_id}/approve`：业务主管审批通过。
- `POST /api/v1/sales/quotations/{quotation_id}/confirm-contract`：客户确认后生成正式出口合同草稿。
- `GET /api/v1/sales/quotations/history`：按客户和产品查询历史报价。
- `GET /api/v1/sales/quotations/purchase-references`：按产品查询采购参考价，优先读取正式采购询价供应商报价，并兼容报价行历史参考价。
- `GET /api/v1/sales/quotations/{quotation_id}/sample-deliveries`：报价页查询关联寄样记录。
- `GET /api/v1/sales/quotations/{quotation_id}/export?format=pdf|excel`：导出预览入口。
- SQLAlchemy async model：`ExportQuotation`、`ExportQuotationLine`。
- Repository 返回 typed row。
- Service 负责权限校验、草稿编辑规则、金额计算、审批状态流转、历史查询、正式采购询价参考、寄样关联、导出和生成正式出口合同。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 出口报价菜单权限：`sales:quotation:view`、`edit`、`view_all`、`approve`、`export`。

前端：

- React 左侧菜单增加“出口报价”。
- React 出口报价页支持列表、搜索、审批状态筛选、客户筛选。
- React 页面支持新增报价、载入草稿、保存草稿编辑、提交审批、审批通过、导出 PDF 预览、客户确认生成正式出口合同。
- React 明细区展示报价抬头、商品明细、历史报价、采购询价参考和寄样关联。
- 业务主管可访问出口报价；财务角色无出口报价菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，便于业务员在报价时同时查看历史价格、采购成本和寄样情况。

## 端到端证据

- 截图：`artifacts/e2e-export-quotations.png`
- E2E 覆盖：
  - 登录业务主管账号。
  - 通过 API 创建前置寄样单，供报价页寄样关联查询。
  - 点击“出口报价”菜单。
  - 等待 `/api/v1/sales/quotations` 返回。
  - 新增出口报价，录入客户、业务员、贸易条款、有效期、商品规格型号、销售单价、货运方式、运费和采购参考价。
  - 载入草稿并保存编辑，等待 `PUT /api/v1/sales/quotations/{quotation_id}`。
  - 提交审批，等待 `/submit`。
  - 审批通过，等待 `/approve`。
  - 校验报价金额、历史报价、采购询价参考和寄样关联显示。
  - 导出 PDF 预览，等待 `/export?format=pdf`。
  - 客户确认生成出口合同，等待 `/confirm-contract`。

## 可用性检查

- 报价列表、状态筛选、客户筛选和新增表单在同一工作区可见。
- 右侧表单覆盖报价抬头和单行报价商品明细，适合当前切片快速录入。
- 明细区集中展示审批状态、金额、报价描述和商品明细。
- 历史报价、采购参考价和寄样关联位于同一明细区，方便报价时横向判断价格和样品情况。
- 草稿编辑、提交审批、审批通过、导出和生成合同按钮按状态启用，减少误操作。
- 财务角色不会看到出口报价菜单，避免越权入口。
