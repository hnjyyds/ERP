# 016 开票通知和税票催收检查记录

## PDF 对照

- `1.5.5.3 开票通知`：从报关单证生成开票通知，按供应商查看开票要求，维护开票品名和数量，发送开票通知，税票催收提醒。
- `docs/05-实现步骤.md 5.3 开票通知`：单证完成后生成供应商开票通知，通知发送后自动提醒业务员催收税票。
- `docs/02-模块设计.md 采购业务`：开票通知属于采购业务核心对象，为供应商税票催收和后续财务往来提供数据。
- `docs/04-数据模型草案.md purchase_invoice_notice`：供应商、报关单证、开票品名、数量、金额、通知日期、状态。

## 本次实现范围

后端：

- `GET /api/v1/purchase/invoice-notices`：开票通知列表，支持文本、状态、供应商、报关单证过滤。
- `POST /api/v1/purchase/invoice-notices/from-customs-declaration`：从报关单证数据生成开票通知，并按供应商分组。
- `GET /api/v1/purchase/invoice-notices/{notice_id}`：开票通知明细。
- `POST /api/v1/purchase/invoice-notices/{notice_id}/send`：发送开票通知，自动生成 7 天后的税票催收提醒。
- `POST /api/v1/purchase/invoice-notices/{notice_id}/receive-tax-invoice`：登记供应商税票，关闭催收提醒。
- `GET /api/v1/purchase/invoice-notices/reminders`：当前用户可见的税票催收提醒列表。
- SQLAlchemy async model：`PurchaseInvoiceNotice`、`PurchaseInvoiceNoticeLine`、`PurchaseInvoiceNoticeReminder`。
- Repository 返回 typed row，封装通知、明细、提醒、状态和统计刷新。
- Service 负责权限校验、供应商分组、发送状态流转、催收提醒生成和收票关闭。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 开票通知菜单权限：`purchase:invoice_notice:view`、`edit`、`view_all`、`send`。

前端：

- 左侧菜单增加“开票通知”。
- 页面支持通知列表、搜索、状态筛选、供应商筛选、报关单证筛选。
- 支持录入报关单证抬头和两组供应商开票要求，从页面生成多条供应商开票通知。
- 明细区展示供应商、报关单、通知金额、开票品名、数量、发送信息、税票号和提醒。
- 支持发送开票通知；发送后页面显示税票催收提醒。
- 支持登记供应商税票；登记后通知状态变为已收票，催收提醒变为已完成。
- 业务主管可访问开票通知；财务角色无开票通知菜单且接口返回 `403`。

  - 校验通知状态为已收票，税票号可见，催收提醒变为已完成。

## 可用性检查

- 列表、筛选、从报关单生成、发送、收票、催收提醒都在同一工作区，适合业务员连续处理。
- 表格优先展示通知号、状态、供应商、报关单和金额，便于扫描。
- 明细区把报关品名和开票品名并列展示，减少开错税票品名的风险。
- 发送和收票按钮按状态禁用，避免重复发送或重复收票。
- 催收提醒在通知明细和全局提醒两处可见，方便业务员按供应商追踪税票。
