## 本次实现范围

后端：

- `GET /api/v1/sample/records`：样品列表，支持文本、样品分类、客户、采购合同过滤。
- `POST /api/v1/sample/records`：新增样品台账，登记初始收样数量和首张图片。
- `GET /api/v1/sample/records/{record_id}`：样品明细、图片、数量事件、库存汇总和跟单节点事件。
- `POST /api/v1/sample/records/{record_id}/images`：追加样品图片元数据。
- `POST /api/v1/sample/records/{record_id}/stock-events`：登记收样/寄样数量事件，并自动计算公司留样数量。
- SQLAlchemy async model：`SampleRecord`、`SampleImage`、`SampleStockEvent`、`SampleStockSummary`、`SampleFollowupEvent`。
- Repository 返回 typed row。
- Service 负责权限校验、分类校验、数量校验、库存汇总和确认样/大货样跟单节点事件发布。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 样品登记菜单权限：`sample:record:view`、`edit`、`view_all`。

前端：

- React 左侧菜单接入“样品登记”。
- React 样品登记页支持列表、搜索、分类筛选、客户筛选、采购合同筛选和新增样品。
- 明细区展示样品来源、客户货号、供应商货号、采购合同、图片、数量汇总、数量事件和采购跟单节点。
- 支持追加样品图片和登记寄样数量。
- 业务主管可访问样品登记；财务角色无样品登记菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，便于业务员和跟单人员持续维护样品台账。

## 端到端证据

- 截图：`artifacts/e2e-sample-records.png`
- E2E 覆盖：
  - 登录业务主管账号。
  - 点击“样品登记”菜单。
  - 等待 `/api/v1/sample/records` 返回。
  - 新增确认样，录入客户货号、供应商货号、采购合同、图片和收样数量。
  - 等待 `/api/v1/sample/records` 201 和列表刷新。
  - 校验确认样生成“确认样提交”采购跟单节点事件。
  - 追加样品图片。
  - 等待 `/api/v1/sample/records/{record_id}/images` 201。
  - 登记寄样数量和寄样单号。
  - 等待 `/api/v1/sample/records/{record_id}/stock-events` 201。
  - 校验寄样数和公司留样数量自动更新。

## 可用性检查

- 样品列表、分类筛选、客户筛选、采购合同筛选和新增表单在同一工作区可见。
- 样品明细集中展示来源、客户货号、供应商货号、采购合同和日期。
- 图片、数量、寄样记录和采购跟单节点位于同一明细区，减少跨页面查询。
- 寄样数量登记后立即更新公司留样数量，便于业务员判断剩余样品。
- 财务角色不会看到样品登记菜单，避免越权入口。
