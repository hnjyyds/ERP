## 本次实现范围

后端：

- `GET /api/v1/purchase/inquiries`：采购询价列表，支持文本、状态、商品、供应商过滤。
- `POST /api/v1/purchase/inquiries`：新增采购询价，包含询价抬头和询价商品明细。
- `PUT /api/v1/purchase/inquiries/{inquiry_id}`：编辑草稿采购询价，替换询价抬头和商品明细；已发模板或已报价后禁止直接编辑。
- `GET /api/v1/purchase/inquiries/{inquiry_id}`：采购询价明细。
- `POST /api/v1/purchase/inquiries/{inquiry_id}/send-template`：生成并标记询价模板发送。
- `POST /api/v1/purchase/inquiries/{inquiry_id}/quotations`：登记供应商报价。
- `GET /api/v1/purchase/inquiries/supplier-samples`：按商品和供应商查询样品证据。
- `GET /api/v1/purchase/inquiries/references`：按商品查询正式采购询价参考价。
- `GET /api/v1/sales/quotations/purchase-references`：已接入正式采购询价报价，同时兼容旧报价行参考价。
- SQLAlchemy async model：`PurchaseInquiry`、`PurchaseInquiryLine`、`SupplierQuotation`。
- Repository 返回 typed row。
- Service 负责权限校验、状态流转、模板内容生成、供应商报价比较、样品证据查询和正式参考价响应。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 采购询价菜单权限：`purchase:inquiry:view`、`edit`、`view_all`、`export`。

前端：

- 左侧菜单增加“采购询价”。
- React 采购询价页支持列表、搜索、状态筛选、商品筛选、供应商筛选。
- 支持新增询价、载入草稿编辑、保存草稿编辑、发送询价模板、登记供应商报价。
- 明细区展示询价商品明细、供应商报价比较、供应商样品关联、正式采购参考价和模板预览。
- 出口报价页“采购询价参考”可读取采购询价中的正式供应商报价。
- 业务主管可访问采购询价；财务角色无采购询价菜单且接口返回 `403`。

- E2E 覆盖：

  - 通过 API 创建供应商样品记录。
  - 点击“采购询价”菜单。
  - 等待 `/api/v1/purchase/inquiries` 返回。
- 新增采购询价，录入业务员、商品、规格型号、数量和备注。
- 载入草稿编辑，修改业务员、询价数量和备注，等待 `PUT /api/v1/purchase/inquiries/{id}` 后回显。
- 发送询价模板，等待 `/send-template`，校验模板预览。
  - 登记两家供应商报价，等待 `/quotations`。
  - 校验供应商报价比较、最低参考价和“有样品”状态。
  - 校验供应商样品关联显示样品编号。
  - 创建出口报价并打开“出口报价”页，校验“采购询价参考”读取本次正式采购询价价格。

## 可用性检查

- 询价列表、筛选、新增表单和明细在同一工作区，采购/业务人员可边查边录。
- 草稿编辑复用新增表单，避免在列表和详情之间来回跳转；已发模板/已报价单据禁用直接编辑，保护供应商报价追踪。
- 询价表单明确区分单头、商品明细和供应商报价，减少字段混淆。
- 供应商报价比较直接展示单价、交期、起订量、样品和备注，适合报价前成本判断。
- 样品关联和采购参考价紧跟供应商报价，减少跨页查询。
- 模板发送返回文本预览，便于后续接入正式邮件或模板系统。
- 财务角色不会看到采购询价菜单，避免越权入口。
