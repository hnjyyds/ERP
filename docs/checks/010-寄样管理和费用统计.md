## 本次实现范围

后端：

- `GET /api/v1/sample/deliveries`：寄样单列表，支持文本、状态、客户、快递公司过滤。
- `GET /api/v1/sample/deliveries`：补充支持寄样日期 `date_from`/`date_to` 范围过滤。
- `POST /api/v1/sample/deliveries`：新增寄样单，包含样品明细和费用明细。
- `PUT /api/v1/sample/deliveries/{delivery_id}`：草稿寄样单编辑，替换抬头、样品明细和费用明细。
- `GET /api/v1/sample/deliveries/{delivery_id}`：寄样单明细。
- `POST /api/v1/sample/deliveries/{delivery_id}/submit`：提交审核。
- `POST /api/v1/sample/deliveries/{delivery_id}/approve`：审核通过，并自动写入样品寄样数量事件。
- `POST /api/v1/sample/deliveries/{delivery_id}/tracking`：更新快递公司、快递单号和物流状态。
- `GET /api/v1/sample/deliveries/fee-statistics`：按客户、日期、快递公司统计寄样费用。
- `GET /api/v1/sample/deliveries/statistics`：按状态、客户、快递公司统计寄样单数和寄样数量。
- `GET /api/v1/sample/deliveries/export`：按客户、日期、快递公司导出寄样 CSV 内容。
- `GET /api/v1/sample/deliveries/history/sample/{sample_record_id}`：查询某样品寄样历史。
- `GET /api/v1/sample/deliveries/quote-history`：按客户和产品查询报价相关寄样历史。
- SQLAlchemy async model：`SampleDelivery`、`SampleDeliveryLine`、`SampleDeliveryFee`。
- Repository 返回 typed row。
- Service 负责权限校验、草稿编辑规则、提交/审核流程、库存充足校验、审核扣减和统计汇总。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 寄样管理菜单权限：`sample:delivery:view`、`edit`、`view_all`、`approve`、`fee:view`。

前端：

- React 左侧菜单增加“寄样管理”。
- React 寄样管理页支持列表、搜索、状态筛选、客户筛选、快递公司筛选。
- React 寄样管理页补充寄样日期范围筛选、寄样 CSV 导出、状态/客户/快递统计表。
- React 页面支持新增寄样单、载入草稿、保存草稿编辑、提交审核、审核通过、物流跟踪更新。
- React 明细区展示寄样抬头、样品明细、费用明细、费用统计、样品寄样历史和报价关联寄样。
- 业务主管可访问寄样管理；财务角色无寄样管理菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，适合业务员连续处理样品寄送、审核和费用核对。

## 端到端证据

- 截图：`artifacts/e2e-sample-deliveries.png`
- E2E 覆盖：
  - 登录业务主管账号。
  - 通过 API 创建前置样品台账，收样数量 5。
  - 点击“寄样管理”菜单。
  - 等待 `/api/v1/sample/deliveries` 返回。
  - 新增寄样单，关联样品、客户、供应商/工厂、报价单和 DHL 费用。
  - 载入草稿并保存编辑，等待 `PUT /api/v1/sample/deliveries/{delivery_id}`。
  - 提交审核，等待 `/submit`。
  - 审核通过，等待 `/approve`。
  - 校验费用统计显示 USD 18.50。
  - 校验样品寄样历史和报价关联寄样显示当前寄样单。
  - 通过样品明细接口校验寄样数 2、公司留样数 3，并确认库存事件关联寄样单号。

## 可用性检查

- 列表、筛选、新增和明细在同一工作区，便于业务员从寄样申请推进到审核。
- 草稿编辑通过“载入编辑 + 保存草稿编辑”完成，避免误改已提交或已审核寄样单。
- 审核动作在明细区直接完成，审核通过后库存和统计立即刷新。
- 费用明细和费用统计并列展示，方便按客户和快递公司核对寄样成本。
- 样品寄样历史和报价关联寄样放在同一明细区，报价时可判断客户是否已经拿过样品。
- 财务角色不会看到寄样管理菜单，避免越权入口。
