## 本次实现范围

后端：

- `POST /api/v1/auth/login`：用户名密码登录，返回 Bearer token、当前用户、权限菜单。
- `GET /api/v1/auth/me`：根据 Bearer token 返回当前用户和菜单。
- `GET /api/v1/system/menus`：根据当前用户权限返回菜单。
- 数据模型：部门、用户、角色、用户角色、权限、角色权限、菜单。
- 密码使用 PBKDF2-HMAC-SHA256 加盐哈希。
- token 使用 HMAC 签名和过期时间校验。
- 工作桌面接口改为从 Bearer token 获取当前用户，不再依赖 `X-User-Id`。

前端：

- 登录页。
- 登录成功保存 token。
- 进入工作桌面后按后端返回菜单渲染左侧导航。
- 侧边栏展示当前用户、部门、角色。
- 支持退出登录。

## 可用性检查

- 登录页聚焦在账号密码输入，不做营销式页面。
- 登录后立即进入工作桌面。
- 侧边栏显示当前用户身份，便于确认权限上下文。
- 菜单数量和角色权限一致，没有越权显示。
- 未登录访问受保护 API 返回 `401`。
