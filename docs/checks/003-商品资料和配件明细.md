## 本次实现范围

后端：

- `GET /api/v1/masterdata/products`：商品列表和编号/中英文名称/海关编码模糊查询。
- `POST /api/v1/masterdata/products`：新增商品，同时写入首批配件明细。
- `GET /api/v1/masterdata/products/{product_id}`：商品明细和配件明细。
- `POST /api/v1/masterdata/products/{product_id}/accessories`：追加商品配件明细。
- `GET /api/v1/masterdata/products/export`：导出商品 CSV 内容。
- SQLAlchemy async model：`Product`、`ProductAccessory`。
- Repository 返回 `ProductRow`、`ProductAccessoryRow` typed row。
- Service 负责权限校验、事务、响应组装和 CSV 导出。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 商品资料菜单权限：`masterdata:product:view`、`masterdata:product:edit`、`masterdata:product:export`。

前端：

- React 左侧菜单显示“商品资料”。
- React 商品资料页支持列表、搜索、新增商品、商品明细、图片地址预览、配件明细、追加配件、导出预览。
- 业务主管可访问商品资料；财务角色无商品资料菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，表格和表单在 1280 宽度无横向溢出。
