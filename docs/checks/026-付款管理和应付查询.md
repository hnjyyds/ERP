## 本次实现范围

后端：

- `POST /api/v1/finance/supplier-invoices`：财务登记供应商发票，并关联已收税票的采购开票通知。
- `GET /api/v1/finance/supplier-invoices`：财务按发票号、供应商、采购合同和付款状态查询供应商发票。
- `POST /api/v1/finance/payment-requests`：财务基于供应商发票创建付款申请，支持预付款、货款和其他类型。
- `GET /api/v1/finance/payment-requests`：财务查询付款申请列表，支持状态、类型、供应商和关键字过滤。
- `POST /api/v1/finance/payment-requests/{payment_request_id}/approve`：财务审批并登记付款金额，生成付款分摊。
- `GET /api/v1/finance/payables`：财务查询供应商应付账款、已付款和应付余额。
- 新增 `modules/finance/payments` 分层模块：model、schema、repository、service、provider。
- 启动时注册付款 ORM 模型，保持 FastAPI controller 只做 HTTP 协调和异常转换。

前端：

- `/finance` 在利润概览和收款工作区下新增付款工作区。
- 供应商发票列表支持搜索、状态筛选、供应商筛选、采购合同筛选、行选择和状态标签。
- 供应商发票登记表单支持发票号、发票日期、供应商、开票通知、采购合同、金额、币种、到期日和备注。
- 付款申请和审批区展示当前供应商发票已付/未付金额，支持新增付款申请和审批付款。
- 付款申请列表展示申请号、状态、类别、供应商、发票号、申请金额和已付金额。
- 应付账款查询支持按关键字、状态、供应商和采购合同号筛选，展示发票金额、已付款、应付余额和付款状态。
- 移动端付款区跟随收款区单列布局，避免页面级横向溢出。
