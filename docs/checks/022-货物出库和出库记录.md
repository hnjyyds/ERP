## 本次实现范围

后端：

- `GET /api/v1/warehouse/outbound-orders`：出库单列表，支持文本、状态、出库模式、出库类型、客户和来源单过滤。
- `POST /api/v1/warehouse/outbound-orders/from-plan`：从出库计划生成出库单，空明细时默认按待出库清单生成。
- `POST /api/v1/warehouse/outbound-orders/{order_id}/submit`：提交出库审批。
- `POST /api/v1/warehouse/outbound-orders/{order_id}/approve`：审批出库，扣减可用库存并写库存流水。
- `GET /api/v1/warehouse/outbound-orders/{order_id}`：出库单详情和明细。
- SQLAlchemy async model：`OutboundOrder`、`OutboundOrderLine`。
- Repository 返回 typed row，封装出库单、明细、库存扣减、库存流水写入。
- Service 负责权限、出库计划校验、正式出库一致性校验、异常出库原因、负库存授权、库存扣减、出库计划回写和采购跟单出库节点回写。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 权限：`warehouse:outbound_order:view/edit/approve/view_all/allow_negative`。

前端：

- 左侧菜单增加“货物出库”。
- 页面支持出库单列表、状态筛选、出库模式筛选、出库类型筛选、客户筛选、来源单筛选。
- 支持从已排库位出库计划生成正式出库或异常出库单。
- 支持提交审批和审批出库。
- 明细区展示来源单据、客户、出库模式、状态、库位、出库商品。
- 同页展示库存余额和出库流水，审批后可立即核对库存扣减。

React 迁移补充：

- `frontend/src/App.tsx` 新增 `/warehouse/outbound-orders` React 路由和 `OutboundOrdersPage`，替代占位模块。
- 复用 Ant Design 表格、表单、按钮、提示和现有 ERP 工作台布局，保持出库登记、审批和库存核对在同一个工作区。
- 新增出库单状态、出库模式、出库类型、来源类型、库存状态和库存流水方向中文标签映射。
- 新增已排库位出库计划选项加载、从出库计划生成出库单、提交审批、审批出库、库存余额查询和库存流水查询的 payload 转换与 API 调用。
- 页面保留 E2E 依赖的字段 ID：`outbound-order-plan-id`、`outbound-order-code`、`outbound-order-mode`、`outbound-order-date`、`outbound-reviewer-name`、`outbound-approved-at` 等。
- `frontend/src/styles.css` 新增出库单概览条、双栏工作区、详情区表格横向保护和移动端单列适配。
- 出库单生成、提交和审批成功提示均在数据刷新后展示，避免用户或 E2E 看到旧库存余额和旧流水。

## 可用性检查

- 出库列表、生成、提交、审批、明细、库存余额和流水在一个工作区内完成。
- 正式出库和异常出库使用显式下拉，异常出库必须填写原因。
- 审批区提供“授权负库存出库”复选项，但后端仍按权限控制。
- 出库计划选择后自动带出计划日期、仓库、库位和经办人，减少重复录入。
- 审批后库存余额和出库流水紧贴出库明细展示，便于仓库人员当场核对。
