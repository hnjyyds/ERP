## 本次实现范围

后端：

- `POST /api/v1/finance/receipts`：财务录入银行水单。
- `GET /api/v1/finance/receipts`：财务按水单号、付款人、客户和状态查询水单。
- `POST /api/v1/finance/receipts/{receipt_id}/claim`：财务或业务员认领待认领水单。
- `POST /api/v1/finance/receipts/{receipt_id}/allocations`：财务把收款分摊到出口合同、发票或预收款。
- `GET /api/v1/finance/receivables`：财务查询已审批出口合同的应收、已收和余额。
- 新增 `modules/finance/receipts` 分层模块：model、schema、repository、service、provider。
- 启动时注册收款 ORM 模型，保持 FastAPI controller 只做 HTTP 协调和异常转换。

前端：

- `/finance` 在原利润概览下新增收款工作区。
- 银行水单列表支持搜索、状态筛选、客户筛选、行选择和状态标签。
- 水单录入表单支持水单号、付款人、客户、金额、币种、银行账号、流水号、收款性质和备注。
- 认领和分摊区展示当前水单已分摊/未分摊金额，支持认领、合同分摊、发票分摊和预收款分摊。
- 应收账款查询支持按合同号和发票号筛选，展示合同金额、已收款、应收余额和收款状态。
- 移动端保持单列工作流，表格横向内容限制在表格容器内部。
