## 本次实现范围

后端：

- `GET /api/v1/sales/shipments`：出货明细列表，支持文本、审批状态、客户、合同过滤。
- `POST /api/v1/sales/shipments/from-contracts`：从一份或多份已审批出口合同生成出货计划。
- `GET /api/v1/sales/shipments/{shipment_id}`：出货明细。
- `POST /api/v1/sales/shipments/{shipment_id}/submit`：提交出货审批。
- `POST /api/v1/sales/shipments/{shipment_id}/approve`：审批通过，回写出口合同已出货数量并刷新合同统计。
- `GET /api/v1/sales/shipments/reminders`：出货提醒列表。
- SQLAlchemy async model：`ShipmentPlan`、`ShipmentLine`。
- Repository 返回 typed row，封装出货列表、明细、财务汇总、审批和提醒查询。
- Service 负责权限校验、已审批合同校验、多合同合并出运、未出货数量校验、收付款利润统计、提醒生成、合同出货进度回写。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 出货菜单权限：`sales:shipment:view`、`edit`、`view_all`、`approve`。

前端：

- 左侧菜单增加“出货明细”。
- React 出货明细页支持列表、搜索、审批状态筛选、客户筛选、合同筛选。
- 支持从出口合同标识生成出货明细，支持两份出口合同一次合并出运。
- 明细区展示出货抬头、物流单证、收付款和利润概览、出货提醒、出货商品明细、合同出货情况、合同采购情况、提醒列表。
- 支持草稿提交审批和待审批单据审批通过。
- 页面沿用 ERP 工作区三栏布局，方便业务员、单证部在同一页面核对出运数据。

- E2E 覆盖：
  - 登录业务主管账号。
  - 通过 API 创建并审批两份出口合同。
  - 点击“出货明细”菜单。
  - 等待 `/api/v1/sales/shipments` 返回。
  - 在页面录入两份合同标识、出货数量、起运港、目的港、船名航次、箱号、订舱号、单证负责人和预计付款成本。

  - 提交审批，等待 `/submit`。
  - 审批通过，等待 `/approve`。
  - 读取出口合同详情接口，确认两份合同已出货数量分别回写为 `550` 和 `200`。

## 可用性检查

- 出货列表、生成表单、明细审批在同一工作区，符合业务员边查边录的流程。
- 表单显式分为出货日期、合同选择、物流单证和预计付款成本，业务部和单证部交接字段清晰。
- 出货明细同步展示应收、付款、利润和利润率，便于主管审批时核对毛利。
- 出货提醒与提醒列表同时展示，既能看当前单据，也能看全局未完成提醒。
- 合同出货和采购情况紧跟出货商品明细，避免审批时跨页核对合同进度。
- 草稿、待审批、已审批按钮按状态启用，减少重复提交和误审批。
- 财务角色不会看到出货明细菜单，接口也会返回 `403`。
