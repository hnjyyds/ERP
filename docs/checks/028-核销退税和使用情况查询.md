## 本次实现范围

后端：

- `POST /api/v1/finance/verification-documents`：财务领用核销单，并自动生成有效期提醒日和提醒消息。
- `GET /api/v1/finance/verification-documents`：财务按关键字、状态、业务员、出运单和提醒状态查询核销单。
- `POST /api/v1/finance/verification-documents/{id}/customs-receipt`：登记报关回单和报关单号。
- `POST /api/v1/finance/verification-documents/{id}/verify`：登记核销编号和核销日期，并完成有效期提醒。
- `POST /api/v1/finance/verification-documents/{id}/tax-refunds`：登记退税金额，校验币种和未退税余额，并更新出运利润。
- `GET /api/v1/finance/verification-usage`：查询核销单使用情况、进度、回单、核销和退税信息。
- 新增 `modules/finance/tax_refunds` 分层模块：model、schema、repository、service、provider。

前端：

- `/finance` 在付费工作区下新增核销退税工作区。
- 核销单列表支持关键字、进度状态、提醒状态、业务员和出运单筛选。
- 核销单领用表单支持核销单号、领用日期、出运单、业务员、客户、有效期、可退税金额和币种。
- 回单、核销和退税登记区展示当前核销单状态、未退税金额和提醒日。
- 核销单使用情况查询支持按关键字、状态、提醒状态和出运单筛选。
- 移动端核销退税区跟随财务页单列布局，避免页面级横向溢出。
