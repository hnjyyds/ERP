## 本次实现范围

后端：

- `GET /api/v1/warehouse/outbound-plans`：出库计划列表，支持文本、状态、出库类型、来源类型、客户、来源单过滤。
- `POST /api/v1/warehouse/outbound-plans/from-shipment`：从已审批发货计划生成成品出库计划。
- `GET /api/v1/warehouse/outbound-plans/{plan_id}`：出库计划详情和待出库明细。
- `POST /api/v1/warehouse/outbound-plans/{plan_id}/schedule`：安排计划出库日期、仓库和库位。
- SQLAlchemy async model：`OutboundPlan`、`OutboundPlanLine`。
- Repository 返回 typed row，封装计划、明细、筛选和排库位持久化。
- Service 负责权限、发货计划审批状态校验、重复生成幂等、明细映射和状态流转。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 权限：`warehouse:outbound_plan:view/edit/view_all`。
- 预留来源类型：`production_requisition`、`processing_issue`；对应生产领料/加工发料模块完成后接入。

前端：

- 左侧菜单增加“出库计划”。
- 页面支持出库计划列表、计划搜索、状态筛选、出库类型筛选、来源类型筛选、客户和来源单筛选。
- 支持从已审批发货计划生成出库计划。
- 支持为选中计划保存计划出库日、仓库、库位和经办人。
- 待出库清单展示来源单据、客户、出库类型、计划状态、计划日期、库位和商品明细。
- 选择发货计划时自动带出计划出库日。

React 迁移补充：

- `frontend/src/App.tsx` 新增 `/warehouse/outbound-plans` React 路由和 `OutboundPlansPage`，替代占位模块。
- 复用 Ant Design 表格、表单、按钮、提示和现有 ERP 工作台布局，保持出库计划的高密度、可扫描工作区。
- 新增出库计划状态、出库类型、来源类型和明细状态中文标签映射。
- 新增已审批发货计划选项加载、从发货计划生成出库计划、保存库位预安排的 payload 转换与 API 调用。
- 页面保留 E2E 依赖的字段 ID：`outbound-plan-shipment-id`、`outbound-plan-type`、`outbound-plan-date`、`outbound-schedule-date`、`outbound-warehouse-id`、`outbound-location-name`、`outbound-operator-name` 等。
- `frontend/src/styles.css` 新增出库计划概览条、双栏工作区、表格横向保护和移动端单列适配。
- 出库计划生成和排库位成功提示均在数据刷新后展示，避免用户或 E2E 看到旧的待出库清单。

## 可用性检查

- 列表、筛选、生成、排库位、待出库清单在一个工作区内完成。
- 状态、来源和出库类型均以中文展示，保留后端稳定状态码。
- 出库计划生成入口只展示已审批发货计划，避免仓库提前处理未审批出运任务。
- 排库位表单使用明确仓库和库位字段，为后续 6.5 货物出库提供可继承数据。
- 待出库清单紧贴计划详情展示，仓库人员可以提前按日期、客户和商品备货。
