## 本次实现范围

后端：

- `GET /api/v1/finance/overview`：财务总览接口。
- 权限：仅 `finance:view` 可访问；业务主管账号访问返回 `403`，未登录返回 `401`。
- 聚合来源：
  - 出货明细：出货单数、应收、应付、预计利润、利润率和近期出运利润明细。
  - 采购开票通知：按状态和币种汇总数量与金额。
  - 样品费用：按付款状态和币种汇总数量与金额。
  - 合作伙伴：启用伙伴数量和伙伴类型汇总。
- 新增 `modules/finance/overview` 分层模块：schema、repository、service、provider。
- FastAPI controller 保持轻量，只处理 token、service 调用和权限异常转换。

前端：

- `/finance` 从占位模块升级为 React 财务工作台页面。
- 页面展示财务概览条、出运利润明细、分币种汇总、采购开票通知状态、样品费用付款状态、合作伙伴费用入口。
- finance 账号从左侧菜单点击“财务管理”后触发 `/api/v1/finance/overview`。
- 移动端保持单列布局，表格在面板内处理横向内容，不再撑宽页面。

## 端到端证据

- 截图：`artifacts/e2e-finance.png`
- E2E 覆盖：
  - 登录业务主管账号。
  - 通过 API 创建出口合同、审批并生成出货明细。
  - 通过 API 创建并发送采购开票通知。
  - 通过 API 创建样品费用并发起付款申请。
  - 通过 API 创建合作伙伴。
  - 退出后登录 finance 账号。
  - 点击“财务管理”菜单并等待 `/api/v1/finance/overview`。
  - 页面显示财务概览、出运利润明细、分币种汇总、开票和付款状态、合作伙伴费用入口。
