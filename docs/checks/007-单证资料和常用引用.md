## 本次实现范围

后端：

- `GET /api/v1/masterdata/document-parties`：单证资料列表，支持编号/名称/客户/国家/联系人/邮箱/电话/银行/SWIFT 模糊查询，支持资料类型和客户标识过滤。
- `POST /api/v1/masterdata/document-parties`：新增单证资料。
- `GET /api/v1/masterdata/document-parties/{party_id}`：单证资料明细。
- `PUT /api/v1/masterdata/document-parties/{party_id}`：编辑单证资料。
- `GET /api/v1/masterdata/document-parties/lookup`：按客户和类型查询可快速引用的有效单证资料。
- SQLAlchemy async model：`DocumentParty`。
- Repository 返回 typed row。
- Service 负责权限校验、资料类型校验、默认项替换、数据权限过滤和响应组装。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 单证资料菜单权限：`masterdata:document_party:view`、`edit`、`view_all`。

前端：

- React 左侧菜单接入“单证资料”。
- React 单证资料页支持列表、搜索、类型过滤、客户过滤、新增、编辑、默认项维护和快速引用查询。
- 资料类型覆盖：收货人、通知人、开证行、提单通知人。
- 业务主管可访问单证资料；财务角色无单证资料菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，便于单证员在重复制作单证时快速选择常用资料。

  - 检查快速引用区能看到同客户同类型资料。
  - 编辑为通知人并更新地址、联系人。

  - 检查列表、详情和引用区可见。

## 可用性检查

- 单证资料列表、类型筛选、客户筛选和新增表单在同一工作区可见。
- 单证资料明细集中展示名称、类型、客户、国家、状态、联系人、默认项和银行信息。
- 快速引用区直接显示同客户同类型可选资料，模拟后续单证选择弹窗的数据来源。
- 类型值在前端显示为业务名称，减少直接暴露系统编码。
- 财务角色不会看到单证资料菜单，避免越权入口。
