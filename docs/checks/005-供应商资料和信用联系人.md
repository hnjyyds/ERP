## 本次实现范围

后端：

- `GET /api/v1/masterdata/suppliers`：供应商列表，支持编号/中英文名称/国家/联系人模糊查询，支持国家和信用等级过滤。
- `POST /api/v1/masterdata/suppliers`：新增供应商，同时写入主联系人和信用信息。
- `GET /api/v1/masterdata/suppliers/{supplier_id}`：供应商明细、联系人、信用信息。
- `PUT /api/v1/masterdata/suppliers/{supplier_id}`：编辑供应商基础资料和信用信息。
- `POST /api/v1/masterdata/suppliers/{supplier_id}/contacts`：追加联系人，可替换主联系人。
- `GET /api/v1/masterdata/suppliers/{supplier_id}/transactions`：供应商交易记录入口，当前返回空列表，后续由采购询价、采购合同、入库、付款接入。
- SQLAlchemy async model：`Supplier`、`SupplierContact`、`SupplierCreditProfile`。
- Repository 返回 typed row。
- Service 负责权限校验、主联系人唯一性、授信额度字段权限、数据权限过滤和响应组装。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 供应商资料菜单权限：`masterdata:supplier:view`、`edit`、`credit:view`、`credit:edit`、`view_all`。

前端：

- React 左侧菜单接入“供应商资料”。
- React 供应商资料页支持列表、搜索、国家/信用等级过滤、新增供应商、编辑供应商、联系人列表、追加联系人、信用信息、交易记录入口。
- 业务主管可访问供应商资料；财务角色无供应商资料菜单且接口返回 `403`。
- 页面沿用 ERP 工作区布局，供业务和采购反复查询维护。

- E2E 覆盖：

  - 点击“供应商资料”菜单。
  - 等待 `/api/v1/masterdata/suppliers` 返回。
  - 新增供应商并写入主联系人和信用信息。
  - 等待 `/api/v1/masterdata/suppliers` 201 和列表刷新。
  - 更新供应商信用等级、授信额度和付款条款。
  - 等待 `/api/v1/masterdata/suppliers/{supplier_id}` PUT 成功。
  - 追加联系人并设置为主联系人。
  - 等待 `/api/v1/masterdata/suppliers/{supplier_id}/contacts` 201。
  - 检查交易记录入口提示可见。

## 可用性检查

- 供应商列表、查询和新增表单在同一工作区可见。
- 供应商明细集中展示名称、国家、状态、信用、授信额度、主联系人和交易记录数量。
- 联系人追加和供应商编辑位于明细区，和新增供应商表单职责分离。
- 表格单元格、筛选条件和按钮文字无重叠。
- 财务角色不会看到供应商资料菜单，避免越权入口。
