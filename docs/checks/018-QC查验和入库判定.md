## 本次实现范围

后端：

- `GET /api/v1/quality/inspections`：QC 查验列表，支持文本、结果、供应商、采购合同过滤。
- `POST /api/v1/quality/inspections`：新增 QC 查验，保存查验明细、异常问题和附件组。
- `PUT /api/v1/quality/inspections/{inspection_id}`：编辑 QC 查验，替换查验明细和异常问题。
- `GET /api/v1/quality/inspections/{inspection_id}`：QC 查验明细和采购合同 QC 历史入口。
- `GET /api/v1/quality/inspections/inbound-eligibility`：按采购合同返回正式入库资格。
- QC 结果包括：通过、不通过、部分通过、待复检。
- QC 通过后自动回写采购跟单 `quality_inspection` 节点实际日期。
- SQLAlchemy async model：`QualityInspection`、`QualityInspectionLine`、`QualityIssue`。
- Repository 返回 typed row，封装查验、明细、异常、最新结果和通过判定。
- Service 负责权限、采购合同审批校验、数据范围、入库资格和跟单回写。
- FastAPI 路由显式声明 `response_model=ApiResponse[...]`。
- 权限：`quality:inspection:view/edit/view_all`。

前端：

- 左侧菜单增加“QC 查验”。
- 页面支持 QC 列表、结果筛选、供应商筛选、采购合同筛选。
- 支持新增和编辑 QC 查验。
- 表单包含采购合同、查验日期、结果、查验人、附件组、查验明细、异常问题和处理措施。
- 明细区展示采购合同、供应商、结果、附件组、入库判定、查验明细和异常问题。
- 支持从页面检查正式入库资格。
- QC 通过保存后，采购跟单页可见 QC 节点实际日期。

React 迁移补充：

- `frontend/src/App.tsx` 新增 `/quality/inspections` React 路由和 `QualityInspectionsPage`，替代占位模块。
- 复用 Ant Design 表格、表单、提示和现有 ERP 工作台布局。
- 新增 QC 结果、异常严重度、问题状态中文标签映射。
- 新增 QC 查验 payload 转换，空值沿用项目统一的 `emptyToNull` 规则。
- 页面保留 E2E 依赖的字段 ID：`quality-code`、`quality-contract-id`、`quality-inspected-at`、`quality-result`、`quality-line-id` 等。
- `frontend/src/styles.css` 新增 QC 页面概览条、双栏工作区、表格横向保护、入库判定提示和移动端适配。

## 端到端证据

- 截图：`artifacts/e2e-quality-inspections.png`
- E2E 覆盖：
  - 登录业务主管账号。
  - 通过 API 创建采购合同，提交并审批。
  - 审批通过后自动生成采购跟单计划。
  - 点击“QC 查验”菜单。
  - 登记 QC 通过记录，包含采购合同明细、查验数量、附件组。
  - 等待 `POST /api/v1/quality/inspections` 返回。
  - 页面显示“QC 节点已回写采购跟单”。
  - 检查正式入库资格，显示“QC 已通过”。
  - 读取采购跟单计划，确认 `quality_inspection` 节点实际日期为 `2026-08-19`。
  - 进入“采购跟单”页面，按采购合同号筛选，确认 QC 节点为已完成。

## 可用性检查

- QC 列表、登记表单、入库判定和历史明细在同一工作区。
- 结果选项使用业务中文展示，避免暴露内部状态码。
- 查验明细单独记录查验数量和不合格数量，便于后续入库比对。
- 异常问题包含严重程度、处理措施和状态，适合 QC 后续追踪。
- 入库资格以明确原因展示，仓库人员后续接入正式入库时可以直接拦截。
