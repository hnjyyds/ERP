# 模块功能补齐蓝图

> 来源：`docs/reference/远景外贸业务管理系统产品介绍-含仓库管理.pdf`、`docs/00-需求范围.md`、`docs/02-模块设计.md`、`docs/checks/*.md` 和当前代码盘点。

## 产品原则

- 首页只做个人工作摘要和跳转入口：待办、提醒、日程、公告、关键 KPI 和快捷入口。大量数据进入专属页面。
- 所有业务模块采用统一页面结构：筛选列表、详情面板、创建大弹窗、编辑大弹窗、删除/作废确认、状态流转、导入导出、附件/日志入口。
- 新增和编辑不能平铺在列表页里；超过 12 个字段的表单使用大弹窗，并按基础信息、业务明细、费用/条款、附件备注、审批信息分区。
- 业务单据优先使用作废/关闭/撤回，不做硬删除；基础资料可停用，已被业务引用的数据必须可追溯。
- 权限遵循后台配置：超级管理员管理组织和公告，业务角色只看到已授权菜单和动作。

## P0 主闭环

P0 先保证外贸业务从接单到结算可完整流转：

1. 商品、客户、供应商、单证资料完整 CRUD。
2. 出口报价、出口合同、采购询价、采购合同完整创建、编辑、提交、审批、作废。
3. 采购跟单、QC 查验、入库计划、入库单、出库计划、出库单完整状态流转。
4. 出货明细、报关/结汇核心单证、开票通知可从上游单据生成并继续维护。
5. 收款、付款、核销退税、结算利润可以登记、审批、反查来源。
6. 经理审批聚合和统计报表能从原始单据下钻。

## 技术落地顺序

1. 建立基础资料 CRUD 样板：商品资料先补齐后端更新/停用/配件维护和前端大弹窗。
2. 抽出通用前端组件：`EntityPageShell`、`FilterToolbar`、`EntityFormModal`、`StatusActionGroup`、`EmptyState`。
3. 按 P0 顺序迁移客户、供应商、单证资料，再迁移销售、采购、仓库、财务。
4. 后端保持分层：router 只负责 schema 和 HTTP 错误映射，service 处理权限/事务/业务规则，repository 只处理数据库。
5. 所有新增接口补单元测试和 schema 测试，文件超过 700 行前拆分。
